## ETL Decisions

### Decision 1 — Date Standardization
Problem: The dataset had inconsistent date formats.
Resolution: All dates were converted into a standard YYYY-MM-DD format before loading.

### Decision 2 — Handling NULL Values
Problem: Some records had missing values in key fields.
Resolution: Missing values were either filled with defaults or removed to maintain data integrity.

### Decision 3 — Category Standardization
Problem: Category names had inconsistent casing (e.g., electronics vs Electronics).
Resolution: All category values were standardized to proper case for consistency.
